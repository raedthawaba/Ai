## تدفق معالجة الطلب (Request Processing Flow)

```
المستخدم
    │
    ▼
┌──────────────────────────────────────────────────────────────────┐
│  API Layer (FastAPI)                                             │
│  POST /api/v1/brain/chat                                         │
└──────────────────────────────┬───────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────────┐
│  HajeenBrain V3.process()                                        │
│                                                                  │
│  ┌─ Step 0 ──────────────────────────────────────────────────┐   │
│  │ Session & Conversation Context Restoration                │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 1 ──────────────────────────────────────────────────┐   │
│  │ Policy Engine                                             │   │
│  │ • فحص محتوى الطلب                                        │   │
│  │ • التحقق من الحصص والأذونات                              │   │
│  │ • إذا محظور → إرجاع استجابة الرفض فوراً                  │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │ (مسموح)                                              │
│  ┌─ Step 2 ──────────────────────────────────────────────────┐   │
│  │ Intent Analyzer (LLM-based)                               │   │
│  │ • استخراج النية الحقيقية بالاستدلال                      │   │
│  │ • تحديد الفئة: CODE / ANALYSIS / PLANNING / ...          │   │
│  │ • استخراج النيات الثانوية والمتطلبات الضمنية             │   │
│  │ • درجة ثقة (0-1)                                         │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 2b ─────────────────────────────────────────────────┐   │
│  │ Goal Manager                                              │   │
│  │ • تحويل النية إلى أهداف قابلة للقياس                     │   │
│  │ • تحديد الأهداف الفرعية                                  │   │
│  │ • تقدير التعقيد والمجال                                  │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 3 ──────────────────────────────────────────────────┐   │
│  │ Context Analyzer                                          │   │
│  │ • استرجاع الذاكرة الدلالية (Cosine Similarity)           │   │
│  │ • تحليل تاريخ المحادثة                                   │   │
│  │ • تحديد المجال ومستوى الخبرة                             │   │
│  │ • تقدير التعقيد والموارد المطلوبة                        │   │
│  │ • تحديد القيود والأولويات                                │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 4 ──────────────────────────────────────────────────┐   │
│  │ Reasoning Engine                                          │   │
│  │ • اختيار استراتيجية الاستدلال (CoT / ToT / ...)          │   │
│  │ • تحليل المشكلة خطوة بخطوة                               │   │
│  │ • اكتشاف المعلومات الناقصة                               │   │
│  │ • تقييم المخاطر                                          │   │
│  │ • مقارنة بدائل الحل واختيار الأفضل                       │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 5 ──────────────────────────────────────────────────┐   │
│  │ Task Decomposer                                           │   │
│  │ Goal → Objectives → Tasks → Actions                       │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 6 ──────────────────────────────────────────────────┐   │
│  │ Graph Planner                                             │   │
│  │ • بناء DAG للمهام                                        │   │
│  │ • تحديد المهام المتوازية والتسلسلية                      │   │
│  │ • Conditional Paths + Retry Policies                      │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 7 ──────────────────────────────────────────────────┐   │
│  │ Decision Engine V3                                        │   │
│  │ • اختيار النموذج (محلي vs سحابي) بالبيانات الفعلية      │   │
│  │ • قرار استخدام RAG / Web Search / Multi-Model            │   │
│  │ • ترتيب المصادر: PerfDB → Registry → Known Quality       │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 8 ──────────────────────────────────────────────────┐   │
│  │ Model Router / Multi-Model Collaborator                   │   │
│  │ • توجيه للنموذج المختار                                  │   │
│  │ • Fallback آلي عند الفشل                                 │   │
│  │ • Ensemble / Voting / Debate إذا قرر ذلك Decision Engine  │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │ (الاستجابة)                                          │
│  ┌─ Step 9 ──────────────────────────────────────────────────┐   │
│  │ Knowledge Distillation                                    │   │
│  │ • استخلاص المعرفة من الاستجابة                           │   │
│  │ • تخزين في Knowledge Graph + Long-term Memory            │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 10 ─────────────────────────────────────────────────┐   │
│  │ Self Reflection                                           │   │
│  │ • تقييم جودة الاستجابة                                   │   │
│  │ • تقييم قرارات النظام                                    │   │
│  │ • اقتراح تحسينات مستقبلية                               │   │
│  └───────────────────────────────────────────────────────────┘   │
│           │                                                      │
│  ┌─ Step 11 ─────────────────────────────────────────────────┐   │
│  │ Sovereignty Layer                                         │   │
│  │ • تسجيل نسبة استخدام النماذج المحلية vs السحابية         │   │
│  │ • تتبع مسيرة الاستقلالية                                 │   │
│  └───────────────────────────────────────────────────────────┘   │
└──────────────────────────────┬───────────────────────────────────┘
                               │
                               ▼
                        استجابة للمستخدم
                        + ExecutionTrace كاملة
```

## تدفق التعلم المستمر (Continuous Learning Flow)

```
بيانات خام من النماذج / المستخدمين
    │
    ▼
ContinuousLearningPipeline.run()
    │
    ├─ Stage 1: Collection
    │   • تحويل Dict → DataSample objects
    │
    ├─ Stage 2: Cleaning
    │   • إزالة الفارغ والقصير (instruction < 5 chars, output < 10 chars)
    │   • تنظيف المسافات
    │
    ├─ Stage 3: Deduplication
    │   • Phase 1: SHA256 hash للكشف السريع عن التطابق الكامل
    │   • Phase 2: Cosine Similarity على Embeddings (threshold: 0.85)
    │   • الاحتفاظ بالعينة ذات الجودة الأعلى عند التكرار
    │
    ├─ Stage 4: Quality Validation
    │   • فلترة العينات بـ quality_score < 0.6
    │
    ├─ Stage 5: Filtering
    │   • كشف المحتوى الضار والمنحاز
    │
    ├─ Stage 6: Ranking
    │   • ترتيب تنازلي حسب quality_score
    │
    ├─ Stage 7: Human Approval (اختياري)
    │   • توقف Pipeline في انتظار موافقة بشرية
    │   • approve_pending() لمتابعة التنفيذ
    │
    ├─ Stage 8: Dataset Builder
    │   • كتابة JSONL: storage_data/brain/learning/dataset_{run_id}.jsonl
    │
    ├─ Stage 9: Training Queue
    │   • التحقق: min 50 عينة
    │   • كتابة Job: storage_data/brain/learning/training_queue.json
    │   • تحديد الأولوية: high إذا avg_quality > 0.8
    │
    ├─ Stage 10: Fine-Tuning
    │   • تحميل النموذج الأساسي (Qwen/Qwen2.5-1.5B)
    │   • تطبيق LoRA (r=16, alpha=32, dropout=0.05)
    │   • تدريب حقيقي بـ HuggingFace Trainer
    │   • إذا لا يوجد GPU: حفظ config لـ Deferred Training
    │
    ├─ Stage 11: Evaluation
    │   • حساب Perplexity الحقيقي
    │   • حساب Token-level Accuracy
    │   • حساب BLEU Score (إن أمكن)
    │   • معيار القبول: perplexity < 30 AND accuracy >= 0.60
    │
    ├─ Stage 12: Deployment (إذا اجتاز التقييم)
    │   • تحديث model_registry.json
    │   • تقاعد النموذج القديم (status: retired)
    │   • كتابة active_model.json
    │
    └─ Stage 13: Rollback (إذا فشل التقييم)
        • تسجيل سبب الفشل
        • الاحتفاظ بالنموذج السابق نشطاً
```

## تدفق Self Reflection (التقييم الذاتي)

```
اكتمال المهمة
    │
    ▼
SelfReflection.reflect(task, result, context)
    │
    ├─ تقييم الخطة: هل كانت مناسبة؟
    ├─ تقييم القرار: هل كان اختيار النموذج صحيحاً؟
    ├─ تقييم الأداء: وقت الاستجابة، الجودة، الكفاءة
    └─ اقتراح التحسينات
         │
         ▼
    حفظ التقرير في storage_data/brain/reflections/
         │
         ▼
SelfEvolution.analyze_and_evolve()
    │
    ├─ تحليل تقارير الانعكاس الأخيرة
    ├─ تحديد أنماط الضعف
    └─ اقتراح تعديلات على السياسات والاستراتيجيات
```

## اختيار النموذج (Model Selection Logic)

```
Decision Engine V3._get_model_quality(model_id)
    │
    ├─ 1. Query ModelPerformanceDB (بيانات فعلية من التشغيل)
    │       └─ إذا وُجد quality_score > 0 → استخدمه ✓
    │
    ├─ 2. Query model_registry.json (النماذج المدرَّبة محلياً)
    │       └─ للنماذج hajeen-*: accuracy × 1.1 → quality ✓
    │
    └─ 3. Known Quality Table (قيم مرجعية)
            ├─ gpt-4o: 0.96
            ├─ claude-3-5-sonnet: 0.95
            ├─ qwen2.5-72b: 0.91
            ├─ qwen2.5-7b: 0.79
            ├─ llama3:70b: 0.88
            └─ unknown: 0.70 (default)
```
