# Hajeen AI - Cognitive Operating System Database Schema

This document outlines the proposed database schema for the Hajeen Cognitive Operating System, focusing on the new components and their data requirements. The schema is designed to be modular, scalable, and compatible with the existing Brain V3 architecture.

## 1. Cognitive Events Table

This table stores the structured cognitive events generated by the Cognitive Compiler.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `event_id`           | UUID      | Unique identifier for the cognitive event          | PRIMARY KEY      |
| `timestamp`          | DATETIME  | Timestamp of the event                             | NOT NULL         |
| `event_type`         | VARCHAR   | Type of cognitive event (e.g., Fact, Concept)      | NOT NULL         |
| `raw_input`          | TEXT      | Original raw input that triggered the event        |                  |
| `goal`               | TEXT      | Objective of the cognitive event                   |                  |
| `intent`             | TEXT      | Underlying intention of the event                  |                  |
| `context`            | JSON      | Contextual information as a JSON object            |                  |
| `knowledge_used`     | JSON      | Knowledge applied during the event                 |                  |
| `models_consulted`   | JSON      | External models or experts consulted               |                  |
| `tools_used`         | JSON      | Tools or APIs utilized                             |                  |
| `thinking_steps`     | TEXT      | Detailed reasoning process followed                |                  |
| `hypotheses`         | JSON      | Generated hypotheses                               |                  |
| `evidence`           | JSON      | Supporting evidence                                |                  |
| `results`            | JSON      | Outcomes of the event                              |                  |
| `errors`             | JSON      | Any errors encountered                             |                  |
| `success_failure_reasons` | TEXT      | Analysis of success or failure                     |                  |
| `confidence_level`   | FLOAT     | System's confidence in the outcome                 |                  |
| `lessons_learned`    | TEXT      | Insights gained from the event                     |                  |

## 2. Concepts Table

This table stores the independent cognitive entities managed by the Concept Engine.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `concept_id`         | UUID      | Unique identifier for the concept                  | PRIMARY KEY      |
| `name`               | VARCHAR   | Name of the concept                                | NOT NULL, UNIQUE |
| `definition`         | TEXT      | Formal definition of the concept                   |                  |
| `properties`         | JSON      | Characteristics and attributes                     |                  |
| `causes`             | JSON      | Factors leading to its existence or changes        |                  |
| `effects`            | JSON      | Consequences or impacts                            |                  |
| `rules`              | JSON      | Governing principles                               |                  |
| `exceptions`         | JSON      | Deviations from rules                              |                  |
| `related_concepts`   | JSON      | Links to other concepts (array of concept_ids)     |                  |
| `evidence`           | JSON      | Supporting data or observations                    |                  |
| `confidence`         | FLOAT     | System's confidence in the concept's validity      |                  |
| `history`            | JSON      | Evolution and changes over time                    |                  |
| `experiences`        | JSON      | Related past experiences (array of event_ids)      |                  |
| `evolution_timeline` | JSON      | Historical timeline of the concept's development   |                  |
| `cognitive_dna_id`   | UUID      | Foreign key to Cognitive DNA table                 | FOREIGN KEY      |

## 3. Cognitive DNA Table

This table stores the cognitive DNA for each concept.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `dna_id`             | UUID      | Unique identifier for the cognitive DNA            | PRIMARY KEY      |
| `concept_id`         | UUID      | Foreign key to Concepts table                      | FOREIGN KEY      |
| `knowledge_source`   | JSON      | Origin of the knowledge                            |                  |
| `num_sources`        | INTEGER   | Quantity of supporting sources                     |                  |
| `source_quality`     | FLOAT     | Reliability and credibility of sources             |                  |
| `confidence_level`   | FLOAT     | System's confidence in the concept                 |                  |
| `change_rate`        | FLOAT     | Frequency of concept updates                       |                  |
| `stability`          | FLOAT     | Consistency over time                              |                  |
| `casual_relationships` | JSON      | Links to cause-and-effect relationships            |                  |
| `associated_experiences` | JSON      | Related experiences (array of event_ids)           |                  |
| `last_updated`       | DATETIME  | Timestamp of the last modification                 |                  |
| `evolution_history`  | JSON      | Record of changes                                  |                  |

## 4. Causal Relationships Table

This table stores the causal relationships discovered by the Knowledge Physics Engine.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `causal_id`          | UUID      | Unique identifier for the causal relationship      | PRIMARY KEY      |
| `cause_concept_id`   | UUID      | Foreign key to Concepts table (cause)              | FOREIGN KEY      |
| `effect_concept_id`  | UUID      | Foreign key to Concepts table (effect)             | FOREIGN KEY      |
| `relationship_type`  | VARCHAR   | Type of causal relationship                        |                  |
| `strength`           | FLOAT     | Strength of the causal link                        |                  |
| `evidence`           | JSON      | Evidence supporting the causal link                |                  |
| `confidence`         | FLOAT     | System's confidence in the causal link             |                  |
| `last_validated`     | DATETIME  | Last time the causal link was validated            |                  |

## 5. Model Performance Table

This table stores performance metrics and metadata for external models used in the Model Society.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `model_id`           | UUID      | Unique identifier for the model                    | PRIMARY KEY      |
| `name`               | VARCHAR   | Name of the model                                  | NOT NULL, UNIQUE |
| `domain`             | VARCHAR   | Specialized domain of the model                    |                  |
| `performance_history` | JSON      | Historical performance data                        |                  |
| `success_rate`       | FLOAT     | Overall success rate                               |                  |
| `common_errors`      | JSON      | Frequently encountered errors                      |                  |
| `confidence_level`   | FLOAT     | System's confidence in the model's output          |                  |
| `cost`               | FLOAT     | Cost per inference/use                             |                  |
| `speed`              | FLOAT     | Inference speed                                    |                  |
| `last_evaluated`     | DATETIME  | Last time the model was evaluated                  |                  |

## 6. Experiments Table

This table stores details about experiments conducted by the Experiment Engine.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `experiment_id`      | UUID      | Unique identifier for the experiment               | PRIMARY KEY      |
| `name`               | VARCHAR   | Name of the experiment                             |                  |
| `description`        | TEXT      | Description of the experiment                      |                  |
| `hypothesis`         | TEXT      | Hypothesis being tested                            |                  |
| `setup`              | JSON      | Experiment setup details                           |                  |
| `results`            | JSON      | Experiment results                                 |                  |
| `conclusion`         | TEXT      | Conclusion drawn from the experiment               |                  |
| `confidence_update`  | JSON      | Impact on confidence levels of related concepts    |                  |
| `start_time`         | DATETIME  | Start time of the experiment                       |                  |
| `end_time`           | DATETIME  | End time of the experiment                         |                  |
| `status`             | VARCHAR   | Current status (e.g., running, completed, failed)  |                  |

## 7. World Model Entities Table

This table stores entities within the World Model.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `entity_id`          | UUID      | Unique identifier for the entity                   | PRIMARY KEY      |
| `name`               | VARCHAR   | Name of the entity                                 | NOT NULL, UNIQUE |
| `type`               | VARCHAR   | Type of entity (e.g., object, agent, location)     |                  |
| `properties`         | JSON      | Attributes of the entity                           |                  |
| `relationships`      | JSON      | Relationships with other entities                  |                  |
| `laws`               | JSON      | Governing laws or rules                            |                  |
| `causes`             | JSON      | Factors influencing the entity                     |                  |
| `effects`            | JSON      | Consequences related to the entity                 |                  |
| `probabilities`      | JSON      | Probabilistic information                          |                  |
| `last_updated`       | DATETIME  | Last time the entity was updated                   |                  |

## 8. Cognitive Versions Table

This table tracks versions of the cognitive system.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `version_id`         | UUID      | Unique identifier for the version                  | PRIMARY KEY      |
| `version_number`     | VARCHAR   | Human-readable version number (e.g., 1.0, 1.1)     | NOT NULL, UNIQUE |
| `changes`            | TEXT      | Description of changes in this version             |                  |
| `reason`             | TEXT      | Reason for the version change                      |                  |
| `evaluation`         | JSON      | Evaluation results of the new version              |                  |
| `rollback_possible`  | BOOLEAN   | Indicates if rollback is possible                  |                  |
| `created_at`         | DATETIME  | Timestamp of version creation                      |                  |
| `parent_version_id`  | UUID      | Foreign key to the previous version                | FOREIGN KEY      |

## 9. Cognitive Constitution Rules Table

This table stores the inviolable rules of the cognitive constitution.

| Field Name           | Data Type | Description                                        | Constraints      |
| :------------------- | :-------- | :------------------------------------------------- | :--------------- |
| `rule_id`            | UUID      | Unique identifier for the rule                     | PRIMARY KEY      |
| `rule_text`          | TEXT      | The rule itself                                    | NOT NULL, UNIQUE |
| `description`        | TEXT      | Detailed description of the rule                   |                  |
| `category`           | VARCHAR   | Category of the rule (e.g., ethics, logic)         |                  |
| `enforcement_mechanism` | TEXT      | How the rule is enforced                           |                  |
| `created_at`         | DATETIME  | Timestamp of rule creation                         |                  |

## 10. Integration with Brain V3

Existing tables and data models within Brain V3 will be extended or linked to these new tables. For instance:

*   **Knowledge Graph (Brain V3):** Will be enhanced by the `Concepts` and `Cognitive DNA` tables.
*   **Memory Fabric (Brain V3):** Will be extended by the `Experience Memory` table.
*   **Model Router (Brain V3):** Will leverage the `Model Performance` table.
*   **Reasoning Engine (Brain V3):** Will be informed by the `Hypothesis Engine` and `Evidence Court`.

Further details on specific integration points will be provided in the Integration Plan.
